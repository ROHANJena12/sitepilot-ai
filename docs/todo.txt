Small future enhancement (not something to implement now)

The only thing I'd consider adding in a future sprint is a report fingerprint for engine versions, for example:

{
  "report_hash": "...",
  "pipeline_version": "pipeline.v1",
  "engine_versions": {
    "seo": "1.0.3",
    "security": "1.2.0",
    "performance": "1.1.5"
  }
}

This would let you know which engine implementations produced a report. If an engine's rules change later, you can tell whether an old report should be regenerated. This is useful for audits over long periods, but it's not necessary right now.
